# LLM Knowledge 与 Agent 系统设计

基于 Karpathy LLM Wiki 加 2025-2026 前沿研究

---

## 1 核心问题 RAG 的局限性

传统 RAG 流程是查询然后检索文档块然后 LLM 合成，每次都重新发现没有积累。

问题包括无跨会话积累，五篇以上文档综合每次重拼，以及向量数据库黑盒难以审计。

核心洞察是知识应该编译一次持续保持最新，而不是每次查询重新派生。

---

## 2 LLM Wiki 架构

### 2.1 核心理念

传统 RAG 是查询时检索每次重新发现，LLM Wiki 是增量编译持续累积。

### 2.2 与 RAG 对比

知识状态方面，RAG 每次重检索而 LLM Wiki 持久化持续更新。

跨会话方面，RAG 无积累而 LLM Wiki 复合增长。

交叉引用方面，RAG 查询时发现而 LLM Wiki 预先建立。

矛盾检测方面，RAG 没有而 LLM Wiki 标记记录。

基础设施方面，RAG 需要向量数据库而 LLM Wiki 使用纯文本 Markdown。

可审计性方面，RAG 是黑盒而 LLM Wiki 完全透明。

### 2.3 三层架构

第一层是原始数据层，存放 PDF 文章 图片等不可变文件，LLM 只读取不修改。

第二层是 Wiki 层，由 LLM 生成的 Markdown 文件组成，包括摘要页 实体页 概念页 对比页，以及 index.md 内容索引和 log.md 操作日志。

第三层是 Schema 层，通过 CLAUDE.md 配置，定义 Wiki 结构 工作流 YAML 规范。

### 2.4 两层核心洞察

第一层是设计目标，建综合 互联 自检的知识网络，而不是平铺索引。

第二层是实现者，让 LLM 承担人类不可行的维护工作。

为什么两层缺一不可，因为人类放弃 Wiki 是因为维护负担增长快于价值，而 LLM 可以不厌倦 不忘记 一次改十五个文件。

### 2.5 Wiki 页面最佳结构

Karpathy 确认的最佳结构依次是摘要 可证伪的断言 交叉引用 矛盾标记。

## 3 操作流程

### 3.1 摄取

添加源文件后，LLM 阅读然后写摘要然后更新索引然后更新实体页概念页，这个过程可能影响十到十五个页面，然后标记矛盾最后追加日志。

### 3.2 查询

在 Wiki 中搜索相关页面，读取后综合答案，答案形式可以是 Markdown 表格或图表，最后附上引用。优质答案应该归档回 Wiki。

### 3.3 整理

定期执行健康检查，包括页面间矛盾检测 过时声明更新 孤立页面连接 缺失概念补全。

## 4 LLM Agent 设计

### 4.1 角色转变

关键分水岭是 2024 年 12 月。之前是百分之八十手写代码加百分之二十 Agent，现在接近零手写代码由 Agent 完成一切。

### 4.2 宏观操作

工程师角色从写代码转变为下达宏观指令和审查结果。Agent 工作模式是多窗口并行，每个任务约二十分钟，屏幕铺开来回切换分配。

### 4.3 AGENTS.md 编写原则

文件开头定义角色，说明 Agent 负责什么任务。

工具集包括文件系统 代码编辑 Git 网络搜索 终端。

记忆工具包括 LLM Wiki 作为外部记忆 会话历史 持久化上下文。

工作流是理解目标然后分解子任务然后并行执行然后综合结果然后验证。

错误处理包括回退策略和何时请求人工介入。

关键洞察是 Agent 表现不佳等于 AGENTS.md 写得不够好，或者未配备记忆工具。

## 5 自动研究实践

### 5.1 核心理念

核心是把人类从循环中移除，设定目标 指标 边界后启动自主运行。

### 5.2 工作流

克隆仓库然后创建分支，然后提出修改包括训练循环 预处理 超参数 架构调整，然后运行五分钟实验然后评估验证损失，如果改进则合并然后循环持续。

### 5.3 program.md 模式

文件描述研究流程，核心是运行五分钟训练实验，用验证损失评估结果，改进则保留否则丢弃，然后提出下一个修改。

每个研究组织本质是一组 Markdown 文件描述角色和流程，不同的 program.md 产生不同的研究进展，可以让模型编写更优的 program.md 实现元优化。

### 5.4 案例 NanoGPT 调优

自动研究运行一晚后发现了手动调优两年未发现的问题，包括 value embeddings 的 weight decay 未调，Adam 的 beta 参数不够精确，以及参数间存在复杂交互效应。

这意味着 AI 研究本身正在被自动化。

## 6 2026 LLM 架构演进

### 6.1 状态空间模型

Mamba 核心创新是选择性状态空间模型和输入依赖参数，优势是 O(n) 线性时间复杂度。

Mamba-2 核心创新是状态空间模型与 Transformer 混合，优势是表达能力更强。

Bamba 由 IBM 开发，核心创新是 Transformer KV-cache 优化，优势是推理速度提升两倍以上匹配精度。

### 6.2 混合专家模型

关键发现是 Top-10 开源模型全部采用混合专家架构，这已成为新标准。

稀疏激活可以解耦容量与计算，100B 总参数 15B 激活参数的效果优于 70B 稠密模型。

NVIDIA GB200 对混合专家模型实现十倍加速，依赖专家并行与硬件协同设计。

混合专家扩展定律有五个关键因素，分别是数据量 总参数 激活参数 激活专家数 共享专家比。最优激活专家数和共享专家比与架构和数据规模无关。当总参数增大时，最优激活参数与总参数之比变得更稀疏。

前沿混合专家模型包括 DeepSeek-R1 Kimi K2 Thinking Mistral Large 3 GPT-OSS-120B，以及 Nemotron 3 Super 该模型总参数 120B 激活参数 12B 采用 Mamba-Attention-混合专家混合架构。

### 6.3 推理优化框架

vLLM 核心特性是分页注意力和连续批处理，支持一百以上并发，适用场景是长上下文 RAG 和灵活部署。

TensorRT-LLM 核心特性是内核融合 FP8 和 INT4 量化以及飞行中批处理，适用场景是最低延迟和最大吞吐量以及 H100 和 B200 芯片。

SGLang 核心特性是基数注意力，吞吐量提升五倍，适用场景是 Agent 多轮对话和树状推理。

### 6.4 投机解码

工作流程是 Draft 模型生成候选 Token 树，Target 模型并行验证，实现数学等价标准解码，加速二到六倍。

EAGLE-3 是 2025 年神经信息处理系统大会的最新工作，使用目标模型大小百分之二到五的草稿头。

### 6.5 量化格式

FP16 或 BF16 格式 7B 模型需要约 14GB 内存，精度损失为基线，适用最大质量场景。

FP8 格式 W8A8 配置需要约 7GB 内存，精度损失小于百分之一，是首选格式和批处理场景。

INT4 AWQ 格式 W4A16 配置需要约 3.5GB 内存，精度损失百分之一到二，适用内存受限场景。

INT4-FP8 格式 W4A8 配置需要约 2GB 内存，精度损失小于百分之二，是最新最优格式。

### 6.6 RAG 2026 前沿

四个演进方向分别是认知语境化 元认知验证 多模态融合 结构化推理。

认知语境化方向论文是 MiA-RAG，核心创新是心智图景和分层递归摘要。

元认知验证方向论文是 Self-RAG，核心创新是反思 Token 和动态检索。

多模态融合方向论文是 RegionRAG，核心创新是区域级检索单元。

结构化推理方向论文是 RAG 加，核心创新是知识与应用示例双语料库。

RAG 加与传统 RAG 对比，传统方式是检索事实然后生成答案，RAG 加方式是检索事实同时检索应用示例然后进行结构化推理，在数学 法律 医疗场景平均提升百分之三到五，复杂场景提升百分之十三点五。

## 8 为什么 Wiki 有效

维护知识库最繁琐的不是阅读或思考，而是记账工作。

人类放弃 Wiki 是因为维护负担增长快于价值增长速度。

LLM 不会厌倦不会忘记忘记更新交叉引用，可以一次修改十五个文件。

维护成本接近零。

人类负责策展源和指导分析和问好问题以及思考意义。

LLM 负责所有繁琐工作包括摘要和交叉引用和归档和簿记，以及维护一致性。
