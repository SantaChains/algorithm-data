# Agents 系统设计

基于 Karpathy LLM Wiki 与 2025-2026 前沿研究

---

## 1 Karpathy-LLM-RAG 架构

### 1.1 三层系统

第一层是原始数据层，存放 PDF 文章 图片等不可变文件，LLM 只读取不修改。

第二层是 Wiki 层，由 LLM 生成的 Markdown 文件组成，包括摘要页 实体页 概念页 对比页，以及 index.md 内容索引和 log.md 操作日志。

第三层是 Schema 层，通过 AGENTS.md 或 CLAUDE.md 配置，定义 Wiki 结构 工作流 错误处理 工具定义。

### 1.2 两层核心洞察

第一层是设计目标，不是建平铺索引，而是建综合 互联 自检的知识网络。

第二层是实现者，让 LLM 承担人类不可行的维护工作。

Wiki 页面最佳结构是四个部分依次为摘要 可证伪断言 交叉引用 矛盾标记。

### 1.3 三类 Agent 系统

Wiki Manager Agent 负责摄取 查询 整理，配置为 CLAUDE.md，核心能力是交叉引用维护和 YAML frontmatter。

Research Agent 负责自动研究 代码优化，配置为 program.md，核心能力是 5 分钟实验循环和 val_bpb 评估。

Home Agent 负责设备控制 安防监控，核心能力是 API 逆向 视觉分析 WhatsApp 推送。

### 1.4 渐进加载阶段

初始建立阶段需要一到两周，配置 AGENTS.md 建立目录结构 建立 wiki 基础页面 配置 index.md 和 log.md。

知识摄取阶段需要三到四周，逐个添加源文档 LLM 生成摘要页 更新实体页概念页 建立交叉引用 标记矛盾点。

查询深化阶段从第二个月开始，复杂问题综合 生成对比分析 创建新页面归档答案 持续丰富 Wiki。

自动维护阶段每月执行，矛盾检测 过时声明更新 孤立页面连接 缺失概念补全。

## 2 2026 前沿技术集成

### 2.1 LLM 架构演进

#### 状态空间模型

Mamba 核心创新是选择性状态空间模型和输入依赖参数，优势是 O(n) 线性时间复杂度。

Mamba-2 核心创新是状态空间模型与 Transformer 混合，优势是表达能力更强。

Bamba 由 IBM 开发，核心创新是 Transformer KV-cache 优化，优势是推理速度提升两倍以上。

状态空间模型与 Transformer 对比：状态空间模型维持压缩隐藏状态，内存开销低推理快。Transformer 是注意力机制，复杂度 O(n²) 但全局建模强。混合架构结合两者优点。

#### 混合专家模型

关键发现是 Top-10 开源模型全部采用混合专家架构，这已成为新标准。

稀疏激活可以解耦容量与计算，100B 总参数 15B 激活参数的效果优于 70B 稠密模型。

NVIDIA GB200 对混合专家模型实现十倍加速，依赖专家并行与硬件协同设计。

混合专家扩展定律有五个关键因素，分别是数据量 总参数 激活参数 激活专家数 共享专家比。最优激活专家数和共享专家比与架构和数据规模无关。当总参数增大时，最优激活参数与总参数之比变得更稀疏。

前沿混合专家模型包括 DeepSeek-R1 Kimi K2 Thinking Mistral Large 3 GPT-OSS-120B，以及 Nemotron 3 Super 该模型总参数 120B 激活参数 12B 采用 Mamba-Attention-混合专家混合架构。

### 2.2 推理优化框架

vLLM 核心特性是分页注意力和连续批处理，支持一百以上并发，适用场景是长上下文 RAG 和灵活部署。

TensorRT-LLM 核心特性是内核融合 FP8 和 INT4 量化以及飞行中批处理，适用场景是最低延迟和最大吞吐量以及 H100 和 B200 芯片。

SGLang 核心特性是基数注意力，吞吐量提升五倍，适用场景是 Agent 多轮对话和树状推理。

### 2.3 核心优化技术

投机解码工作流程是 Draft 模型生成候选 Token 树，Target 模型并行验证，实现数学等价标准解码，加速二到六倍。EAGLE-3 是 2025 年神经信息处理系统大会的最新工作，使用目标模型大小百分之二到五的草稿头。

量化格式对比：FP16 或 BF16 格式 7B 模型需要约 14GB 内存，精度损失为基线，适用最大质量场景。FP8 格式 W8A8 配置需要约 7GB 内存，精度损失小于百分之一，是首选格式和批处理场景。INT4 AWQ 格式 W4A16 配置需要约 3.5GB 内存，精度损失百分之一到二，适用内存受限场景。INT4-FP8 格式 W4A8 配置需要约 2GB 内存，精度损失小于百分之二，是最新最优格式。

KV Cache 优化包括分页注意力将内存浪费从百分之六十到八十降低到百分之四以下，前缀缓存复用共同前缀，FP8 或 INT8 KV 实现二到三倍内存节省。

### 2.4 RAG 2026 前沿架构

四个演进方向分别是认知语境化 元认知验证 多模态融合 结构化推理。

认知语境化方向论文是 MiA-RAG，核心创新是心智图景和分层递归摘要。

元认知验证方向论文是 Self-RAG，核心创新是反思 Token 和动态检索。

多模态融合方向论文是 RegionRAG，核心创新是区域级检索单元。

结构化推理方向论文是 RAG 加，核心创新是知识与应用示例双语料库。

RAG 加与传统 RAG 对比：传统方式是检索事实然后生成答案，RAG 加方式是检索事实同时检索应用示例然后进行结构化推理，在数学 法律 医疗场景平均提升百分之三到五，复杂场景提升百分之十三点五。

TREC 2025 RAG 轨道最佳实践是检索使用 BM25 和密集融合然后使用排名融合，重排使用后期交互方法，生成使用分层两级引用即子查询答案到综合答案，评估使用联合金块覆盖率和句子支持率。

## 3 Agent 工作模式

### 3.1 并行 Agent 布局

屏幕布局采用四个 Agent 并行工作，第一个 Agent 负责写代码，第二个 Agent 负责做研究，第三个 Agent 负责规划，第四个 Agent 负责审查，每个 Agent 执行约二十分钟任务后切换分配。

宏观操作是下达指令加审查结果，与微观操作的逐行代码方式不同。

### 3.2 失败排查清单

首先检查 AGENTS.md 配置是否正确，包括角色定义是否清晰 工具集是否完整 工作流是否明确。

然后检查记忆工具是否配置，包括 LLM Wiki 是否初始化 上下文是否足够。

最后检查任务分解是否合理，包括子任务是否过于复杂 验证标准是否明确。

### 3.3 多层记忆架构

第一层是工作记忆，存储当前会话上下文包括最近几轮对话 当前任务状态和临时变量。

第二层是会话记忆，存储会话级持久化包括会话摘要 关键决策记录和中间结果。

第三层是 Wiki 记忆，存储长期知识库包括实体页 概念页和交叉引用。

## 4 工具速查

AGENTS.md 或 CLAUDE.md 用于 Agent 配置和行为规范。

program.md 用于自动研究研究流程。

raw 目录存放源文档，wiki 目录存放 LLM 生成知识。

index.md 是内容索引，log.md 是操作日志。

Obsidian 和 qmd 分别用于 Wiki 可视化和本地搜索。

vLLM TRT-LLM SGLang 用于推理部署。

### 4.1 重要开源实现

Understand-Anything 项目获得九千五百颗星，特点是交互式知识图谱可探索任意代码库和 Wiki。

llmwiki 项目获得七百三十二颗星，特点是 Next.js 加 FastAPI 加 Supabase 加 MCP。

karpathy-llm-wiki 项目获得六百六十七颗星，特点是 Agent Skills 兼容支持 Claude Code 和 Cursor 和 Codex。

obsidian-llm-wiki-local 项目获得四百一十三颗星，特点是百分之百本地运行使用 Ollama。

autodidact-skill 项目特点是自己动的研究 Agent。
