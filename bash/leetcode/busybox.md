## 命令规范

### awk

**用途：** 文本处理和模式扫描的语言

**语法：**
```bash
awk 'pattern {action}' file
awk -F '分隔符' 'pattern {action}' file
```

**常用选项：**
| 选项 | 说明 |
|------|------|
| `-F` | 指定字段分隔符 |
| `-v` | 设置变量 |

**内置变量：**
| 变量 | 说明 |
|------|------|
| `NF` | 当前行的字段数 |
| `NR` | 当前记录的序号（行号） |
| `FS` | 字段分隔符（默认空格） |
| `RS` | 记录分隔符（默认换行） |
| `$0` | 整行内容 |
| `$1,$2...$n` | 第 n 个字段 |
| `$NF` | 最后一个字段 |

**常用模式：**
```bash
# 按行号处理
awk 'NR==10' file          # 第10行
awk 'NR>=5 && NR<=10' file # 第5-10行
awk 'NR>1' file            # 跳过第一行

# 按字段处理
awk '{print $2}' file      # 打印第2列
awk '$3>100' file          # 第3列大于100的行

# 统计计算
awk '{sum+=$1} END{print sum}' file
awk '{for(i=1;i<=NF;i++)count[$i]++} END{for(k in count) print k,count[k]}' file
```

### sed

**用途：** 流编辑器，用于文本替换、删除、插入等

**语法：**
```bash
sed [选项] '命令' file
sed -e 'cmd1' -e 'cmd2' file
```

**常用选项：**
| 选项 | 说明 |
|------|------|
| `-n` | 抑制自动输出，仅显示处理结果 |
| `-i` | 直接修改文件 |
| `-e` | 多重编辑 |
| `-r` | 使用扩展正则 |

**常用命令：**
```bash
# 打印
sed -n '10p' file          # 打印第10行
sed -n '1,5p' file         # 打印1-5行
sed -n '5,$p' file         # 打印第5行到最后

# 替换
sed 's/old/new/g' file     # 全局替换
sed 's/old/new/2' file     # 每行只替换第2个
sed '10s/old/new/' file    # 只替换第10行

# 删除
sed '10d' file             # 删除第10行
sed '1,5d' file            # 删除1-5行
sed '/pattern/d' file      # 删除匹配的行

# 插入
sed '1i\header' file      # 在第1行前插入
sed '1a\add text' file    # 在第1行后追加
```

**正则示例：**
```bash
sed -E 's/[0-9]{3}/XXX/g' file  # 隐藏数字
sed -E 's/(.*),(.*)/\2 \1/' file # 交换两个逗号分隔的字段
```

### 文件操作命令

#### ls - 列出目录内容
```bash
ls [选项] [文件...]
ls -l          # 详细列表
ls -a          # 包含隐藏文件
ls -lh         # 人性化大小
ls -1          # 每行一个
```

#### cd - 切换目录
```bash
cd [目录]      # 切换到指定目录
cd ~           # 切换到主目录
cd -           # 切换到上一个目录
cd ..          # 返回上一级
```

#### mkdir - 创建目录
```bash
mkdir [选项] 目录
mkdir -p a/b/c    # 递归创建
mkdir -m 755 dir  # 指定权限
```

#### rm - 删除文件/目录
```bash
rm [选项] 文件
rm -r dir        # 递归删除目录
rm -f file       # 强制删除
rm -i file       # 交互式确认
```

#### cp - 复制文件/目录
```bash
cp [选项] 源 目标
cp -r dir1 dir2     # 递归复制目录
cp -i file1 file2   # 覆盖前询问
cp -p file1 file2   # 保留属性
```

#### mv - 移动/重命名
```bash
mv [选项] 源 目标
mv -i file1 file2   # 覆盖前询问
mv oldname newname  # 重命名
```

#### cat - 显示文件内容
```bash
cat [选项] 文件
cat -n file         # 显示行号
cat -s file         # 压缩空行
cat file1 file2     # 合并文件
```

#### touch - 创建空文件/更新时间戳
```bash
touch [选项] 文件
touch file          # 创建空文件
touch -t 20231225 file  # 指定时间
touch -a file       # 只更新访问时间
touch -m file       # 只更新修改时间
```

#### nl - 编号行
```bash
nl [选项] 文件
nl -ba file         # 所有行都编号
nl -w 5 file        # 行号宽度5
nl -s '  ' file     # 分隔符
```

## 192. 统计词频

**难度：** 中等 | **状态：** 已解答

### 题目描述

写一个 bash 脚本以统计一个文本文件 `words.txt` 中每个单词出现的频率。

假设：

- `words.txt` 只包括小写字母和空格
- 每个单词只由小写字母组成
- 单词间由一个或多个空格字符分隔

### 示例

**输入文件 `words.txt`：**

```
the day is sunny the the
the sunny is is
```

**预期输出（以词频降序排列）：**

```
the 4
is 3
sunny 2
day 1
```

### 说明

词频相同的单词顺序任意，每个单词出现的频率都是唯一的。可以使用一行 Unix pipes 实现。

### 解法一（推荐）

```bash
awk '{for(i=1;i<=NF;i++) w[$i]++} END{for(k in w) print k,w[k]}' words.txt | sort -nrk2
```

**复杂度分析：**
| 指标 | 复杂度 | 说明 |
|------|--------|------|
| 时间 | O(n × w × log w) | n行×每行词数w，排序log w |
| 空间 | O(u) | u为不重复单词数 |
| 内存 | 中 | 哈希表存储所有单词 |

**优化点：** 单次遍历完成计数，无需中间文件

---

### 解法二

```bash
cat words.txt | tr -s ' ' '\n' | sort | uniq -c | sort -nr | awk '{print $2, $1}'
```

**复杂度分析：**
| 指标 | 复杂度 | 说明 |
|------|--------|------|
| 时间 | O(n × w × log n) | 多次排序 |
| 空间 | O(n) | sort需要临时存储 |
| 内存 | 较高 | 多次磁盘/内存交换 |

**优化点：** 思路直观，但多次排序效率较低

---

### 解法对比

| 解法 | 时间 | 空间 | 内存 | 推荐场景 |
|------|------|------|------|----------|
| awk哈希计数 | O(n·w·log u) | O(u) | ★★★★★ | 通用首选 |
| sort+uniq链式 | O(n·w·log n) | O(n) | ★★☆☆☆ | 数据量大时 |

**性能建议：**
- awk解法更优，单次遍历+O(n)空间
- 避免 `cat file | ...` 开头的多余管道
- 大文件可加 `| head` 限制输出行数

## 193. 有效电话号码

**难度：** 简单 | **状态：** 已解答

### 题目描述

给定一个包含电话号码列表（一行一个电话号码）的文本文件 `file.txt`，写一个单行 bash 脚本输出所有有效的电话号码。

有效电话号码必须满足以下两种格式之一：

- `(xxx) xxx-xxxx`
- `xxx-xxx-xxxx`

（x 表示一个数字）

假设每行前后没有多余的空格字符。

### 示例

**输入文件 `file.txt`：**

```
987-123-4567
123 456 7890
(123) 456-7890
```

**预期输出：**

```
987-123-4567
(123) 456-7890
```

### 解法

```bash
grep -E '^([0-9]{3}-[0-9]{3}-[0-9]{4}|\([0-9]{3}\) [0-9]{3}-[0-9]{4})$' file.txt
```

**复杂度分析：**
| 指标 | 复杂度 | 说明 |
|------|--------|------|
| 时间 | O(n × L) | n行×每行长度L，正则匹配 |
| 空间 | O(1) | 仅存储当前匹配结果 |
| 内存 | ★★★★★ | 流式处理，无内存累积 |

**正则解析：**
| 部分 | 匹配 |
|------|------|
| `[0-9]{3}` | 3位数字（xxx） |
| `[0-9]{3}-[0-9]{3}-[0-9]{4}` | xxx-xxx-xxxx 格式 |
| `\([0-9]{3}\) [0-9]{3}-[0-9]{4}` | (xxx) xxx-xxxx 格式 |
| `^...$` | 整体匹配，行首到行尾 |

**性能建议：**
- `grep -E` 使用扩展正则，效率高
- 避免 `cat file | grep` 多余管道
- 大文件可加 `-l` 仅返回文件名（多文件时）

## 194. 转置文件

**难度：** 中等 | **状态：** 已解答

### 题目描述

给定一个文件 `file.txt`，转置它的内容。

假设每行列数相同，并且每个字段由空格分隔。

### 示例

**输入文件 `file.txt`：**

```
name age
alice 21
ryan 30
```

**预期输出：**

```
name alice ryan
age 21 30
```

### 解法

```bash
awk '{for(i=1;i<=NF;i++)a[i]=a[i](NR>1?" ":"")$i}END{for(i=1;i<=NF;i++)print a[i]}' file.txt
```

**复杂度分析：**
| 指标 | 复杂度 | 说明 |
|------|--------|------|
| 时间 | O(R × C) | R行×C列，完整遍历 |
| 空间 | O(R × C) | 需存储所有单元格 |
| 内存 | 中高 | a数组存储完整转置矩阵 |

**算法解析：**
1. `NR>1?" ":""` — 第一列不添加前导空格
2. `a[i]=a[i]...$i` — 按列累积字符串
3. `END{for(i=1;i<=NF;i++)print a[i]}` — 按列输出

**优化建议：**
- 转置是固有O(R×C)操作，无法优化时间
- 可用 `BEGIN{FS=OFS="\t"}` 处理制表符分隔
- 大矩阵文件考虑 `awk -f script.awk file` 外置脚本

## 195. 第十行

**难度：** 简单 | **状态：** 已解答

### 题目描述

给定一个文本文件 `file.txt`，请只打印该文件中的第十行。

### 示例

**输入文件 `file.txt`：**

```
Line 1
Line 2
Line 3
Line 4
Line 5
Line 6
Line 7
Line 8
Line 9
Line 10
```

**预期输出：**

```
Line 10
```

### 说明

- 如果文件少于十行，应当输出什么？
- 至少用三种不同的解法实现。

### 解法一（推荐）

```bash
sed -n '10p' file.txt
```

**复杂度分析：**
| 指标 | 复杂度 | 说明 |
|------|--------|------|
| 时间 | O(n) | 需读取前10行，n为文件总行数 |
| 空间 | O(1) | 仅缓存当前行 |
| 内存 | 低 | 无需加载整个文件 |

**优化点：** `sed` 是流式处理，找到第10行后立即退出，不会继续读取

---

### 解法二

```bash
awk 'NR==10' file.txt
```

**复杂度分析：**
| 指标 | 复杂度 | 说明 |
|------|--------|------|
| 时间 | O(n) | 需遍历到第10行 |
| 空间 | O(1) | 仅存储计数器NR |
| 内存 | 极低 | awk内部变量开销小 |

---

### 解法三

```bash
head -n 10 file.txt | tail -n 1
```

**复杂度分析：**
| 指标 | 复杂度 | 说明 |
|------|--------|------|
| 时间 | O(n) | 需读取前10行 |
| 空间 | O(1) | 管道流式处理 |
| 内存 | 低 | 两个命令协同工作 |

**优化点：** 管道方式，命令退出后立即释放内存

---

### 解法四

```bash
tail -n +10 file.txt | head -n 1
```

**复杂度分析：**
| 指标 | 复杂度 | 说明 |
|------|--------|------|
| 时间 | O(n) | tail需跳到最后或seek |
| 空间 | O(1) | 流式处理 |
| 内存 | 低 | 只需一行缓冲 |

**注意：** `tail -n +10` 从第10行开始输出，配合 `head -n 1` 取第一行

---

### 解法对比

| 解法 | 时间 | 空间 | 内存 | 推荐场景 |
|------|------|------|------|----------|
| sed -n '10p' | O(n) | O(1) | ★★★★★ | 通用首选 |
| awk 'NR==10' | O(n) | O(1) | ★★★★★ | 熟悉awk时 |
| head+tail | O(n) | O(1) | ★★★★☆ | 命令组合 |
| tail+head | O(n) | O(1) | ★★★☆☆ | 文件中部行 |

**性能建议：**
- 大文件场景优先 `sed -n '10p'` 或 `awk 'NR==10'`
- 避免 `cat file.txt | ...` 多余管道
- `tail -n +10 file.txt | head -1` 对大文件更友好（seek而非读取）
